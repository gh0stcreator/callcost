# ТЗ: Meeting Cost — счётчик стоимости Zoom-звонка

**Версия:** 1.0  
**Статус:** Готово к реализации  
**Что уже сделано:** UI, логика подсчёта, приватная модель данных, Zoom SDK интеграция  
**Что остаётся:** Хостинг + регистрация в Zoom Marketplace + (опционально) Firebase для кросс-устройственной синхронизации

---

## 1. Суть продукта

Панель внутри Zoom-звонка (боковой сайдбар, ~360px). Каждый участник при входе вводит свою месячную зарплату — **приватно, никто другой её не видит**. Приложение считает стоимость звонка в реальном времени: большой тикающий счётчик как в такси, плюс анонимный вклад каждого участника.

---

## 2. Пользовательский сценарий

### Первый вход
1. Участник открывает панель Meeting Cost внутри Zoom
2. Видит экран с замком: «Сколько ты получаешь?»
3. Вводит зарплату за месяц (например, 150 000 ₽) и количество рабочих дней (по умолчанию 21)
4. Приложение показывает формулу: `150 000 ÷ 21 ÷ 8 ÷ 3600 = 0.248 ₽/сек`
5. Нажимает «Войти в звонок» → переходит на экран счётчика

### Основной экран
- Большой тикающий тотал (сумма всех участников)
- Под ним: «Твой вклад: ₽X.XX — только ты это видишь»
- Счётчик времени + количество участников
- Кнопки: Старт / Пауза / Сброс
- Список анонимных участников: «Участник 1 — ₽12.34», «Участник 2 — ₽9.87»

### Повторный вход
- Зарплата сохранена в localStorage — сразу попадает на экран счётчика
- Ссылка «↺ Изменить мою зарплату» внизу панели

---

## 3. Модель приватности (критично)

| Данные | Где хранится | Кому видны |
|--------|-------------|------------|
| Сумма зарплаты | localStorage браузера | Только владелец |
| Секундная ставка (₽/сек) | Вычисляется локально | Только владелец |
| Текущая сумма участника | Broadcast-канал (анонимно) | Все — но без имени и ставки |
| Тотал звонка | Вычисляется каждым клиентом | Все |

**Принцип:** по каналу синхронизации идут только анонимные числа вида `{ uuid: "u_x7k3m", cost: 12.34 }`. Имя, зарплата и ставка в канал не попадают никогда.

---

## 4. Формула расчёта

```
секундная_ставка = зарплата_в_месяц ÷ рабочие_дни ÷ 8_часов ÷ 3600_секунд

вклад_участника = секундная_ставка × прошедшее_время_в_секундах

тотал = сумма вкладов всех участников
```

Пример при ЗП 120 000 ₽, 21 день, звонок 30 минут, 4 участника с одинаковой ЗП:
```
ставка = 120000 / 21 / 8 / 3600 = 0.198 ₽/сек
вклад каждого за 30 мин = 0.198 × 1800 = 357 ₽
тотал = 357 × 4 = 1 428 ₽
```

## 5. Технический стек

| Компонент | Решение | Почему |
|-----------|---------|--------|
| Frontend | HTML + Vanilla JS | Без сборки, легко деплоить |
| Шрифты | Syne (UI) + DM Mono (счётчик) | Google Fonts, бесплатно |
| Zoom интеграция | Zoom Apps SDK (`appssdk.zoom.us/sdk.js`) | Официальный SDK |
| Синхронизация (базовая) | BroadcastChannel API | Работает между вкладками одного браузера |
| Синхронизация (полная) | Firebase Realtime Database | Кросс-устройственная, нужна для реального использования |
| Хостинг | Vercel (бесплатно) | HTTPS из коробки, нужен Zoom |
| Данные пользователя | localStorage | Только на устройстве, не передаётся |

## 7. Синхронизация: два уровня

### Уровень 1 — BroadcastChannel (уже реализован)
- Работает между вкладками **одного браузера**
- Нулевая настройка, не нужны ключи
- Подходит для демо и для хоста, который шарит экран

### Уровень 2 — Firebase (нужно добавить для продакшна)
- Работает между **разными устройствами и браузерами**
- Каждый участник открывает панель со своего устройства — все видят один счётчик
- Нужно: создать Firebase-проект → получить конфиг (7 строк) → вставить в index.html

**Что передаётся в Firebase:**
```json
{
  "room_ID123": {
    "timer": { "running": true, "startTs": 1712345678000, "elapsed": 0 },
    "peers": {
      "u_x7k3m": { "cost": 12.34, "lastSeen": 1712345690000 },
      "u_p9q2r": { "cost": 9.87, "lastSeen": 1712345689000 }
    }
  }
}
```
Зарплата и ставка в Firebase **не попадают**.

## 8. Zoom App: что и где настраивать

### 8.1 Регистрация на marketplace.zoom.us

| Поле | Значение |
|------|----------|
| App Type | General App |
| Surface | In-Meeting App |
| Home URL | `https://your-app.vercel.app/index.html` |
| Redirect URL | `https://your-app.vercel.app/index.html` |

### 8.2 Scopes (разрешения)
```
meeting:read:meeting_participants:local
meeting:read:meeting_context:local
```

### 8.3 Режим публикации
- **Для себя/команды:** нажать «Add» без Submit for Review — работает сразу
- **Для всех:** Submit for Review (Zoom проверяет 2–4 недели)

### 8.4 Что делает Zoom SDK в коде
```javascript
await zoomSdk.config({
  capabilities: ['getMeetingParticipants', 'getMeetingContext'],
  version: '0.16'
});
// После этого можно получить список участников
const res = await zoomSdk.getMeetingParticipants();
// res.participants[].screenName — имена из Zoom (только для отображения, не для ставок)
```

## 9. CSP-заголовки (vercel.json)

Zoom требует, чтобы приложение разрешало встраивание в iframe:

```json
{
  "headers": [{
    "source": "/(.*)",
    "headers": [
      {
        "key": "Content-Security-Policy",
        "value": "frame-ancestors 'self' https://*.zoom.us https://zoom.us"
      }
    ]
  }]
}
```

Без этого Zoom не загрузит панель.

## 12. Глоссарий

| Термин | Объяснение |
|--------|------------|
| BroadcastChannel | Браузерный API для обмена сообщениями между вкладками одного домена |
| Zoom Apps SDK | JavaScript-библиотека Zoom для работы внутри Zoom-клиента |
| localStorage | Хранилище браузера на устройстве пользователя, недоступное извне |
| Firebase Realtime DB | База данных Google с синхронизацией в реальном времени |
| CSP / frame-ancestors | Политика безопасности контента: кто может встраивать страницу в iframe |
| uuid | Случайный анонимный идентификатор участника (например, `u_x7k3m2p`) |
